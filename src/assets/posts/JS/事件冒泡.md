# 深入理解 JavaScript 事件冒泡与事件捕获

## 什么是事件冒泡？

事件冒泡（Event Bubbling）是 DOM 事件传播的一种机制，当一个元素上的事件被触发时，该事件会从触发元素开始，逐级向上传播到其父元素、祖父元素，直到到达文档根节点。

## 事件传播的三个阶段

DOM 事件传播分为三个阶段：

1. **捕获阶段（Capture Phase）**：事件从文档根节点向下传播到目标元素
2. **目标阶段（Target Phase）**：事件到达目标元素
3. **冒泡阶段（Bubbling Phase）**：事件从目标元素向上传播回文档根节点

```

├──────────────────────────────────────────────────────────────┤
```

## 事件冒泡示例

```html
<div id="outer">
  <div id="middle">
    <div id="inner">点击我</div>
  </div>
</div>
```

```javascript
const outer = document.getElementById('outer');
const middle = document.getElementById('middle');
const inner = document.getElementById('inner');

outer.addEventListener('click', () => console.log('outer clicked'));
middle.addEventListener('click', () => console.log('middle clicked'));
inner.addEventListener('click', () => console.log('inner clicked'));

// 点击 inner，输出顺序：
// inner clicked
// middle clicked
// outer clicked
```

## 事件捕获

事件捕获（Event Capture）是事件传播的另一种方式，与事件冒泡相反，事件从文档根节点开始，逐级向下传播到目标元素。

### 使用事件捕获

```javascript
outer.addEventListener('click', () => console.log('outer captured'), true);
middle.addEventListener('click', () => console.log('middle captured'), true);
inner.addEventListener('click', () => console.log('inner captured'), true);

// 点击 inner，输出顺序：
// outer captured
// middle captured
// inner captured
```

### 混合使用捕获和冒泡

```javascript
outer.addEventListener('click', () => console.log('outer captured'), true);
outer.addEventListener('click', () => console.log('outer bubbled'));
middle.addEventListener('click', () => console.log('middle bubbled'));
inner.addEventListener('click', () => console.log('inner'));

// 点击 inner，输出顺序：
// outer captured
// inner
// middle bubbled
// outer bubbled
```

## 阻止事件冒泡

有时我们需要阻止事件继续传播，可以使用以下方法：

### stopPropagation()

```javascript
inner.addEventListener('click', (e) => {
  console.log('inner clicked');
  e.stopPropagation(); // 阻止事件继续传播
});

middle.addEventListener('click', () => console.log('middle clicked'));

// 点击 inner，只会输出：inner clicked
```

### stopImmediatePropagation()

阻止事件继续传播，并且阻止同一元素上的其他事件监听器被调用：

```javascript
inner.addEventListener('click', (e) => {
  console.log('first listener');
  e.stopImmediatePropagation();
});

inner.addEventListener('click', () => {
  console.log('second listener'); // 不会被执行
});

// 点击 inner，只会输出：first listener
```

## 事件委托

事件委托是一种利用事件冒泡机制实现的设计模式，通过将事件监听器添加到父元素上，来处理子元素的事件。

### 传统方式

```javascript
// 需要为每个列表项添加事件监听器
const items = document.querySelectorAll('li');
items.forEach(item => {
  item.addEventListener('click', () => {
    console.log(item.textContent);
  });
});
```

### 事件委托方式

```javascript
// 只需要为父元素添加一个事件监听器
const list = document.getElementById('list');
list.addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    console.log(e.target.textContent);
  }
});
```

### 事件委托的优势

1. **性能更好**：减少事件监听器的数量
2. **动态元素**：新添加的子元素自动继承事件处理
3. **代码简洁**：统一的事件处理逻辑

## 实际应用场景

### 场景 1：表单验证

```javascript
const form = document.getElementById('myForm');
form.addEventListener('blur', (e) => {
  const target = e.target;
  if (target.tagName === 'INPUT') {
    validateInput(target);
  }
}, true); // 使用捕获阶段
```

### 场景 2：模态框关闭

```javascript
const modal = document.getElementById('modal');
modal.addEventListener('click', (e) => {
  if (e.target === modal) {
    modal.style.display = 'none';
  }
});
```

### 场景 3：无限滚动

```javascript
const container = document.getElementById('scrollContainer');
container.addEventListener('scroll', (e) => {
  const target = e.target;
  if (target.scrollTop + target.clientHeight >= target.scrollHeight) {
    loadMoreData();
  }
});
```

## 常见误区

### 误区 1：所有事件都会冒泡

实际上，并不是所有事件都会冒泡。以下事件不会冒泡：

- `focus`
- `blur`
- `scroll`
- `resize`

### 误区 2：事件冒泡总是从目标元素开始

实际上，事件冒泡是从事件触发的元素开始，而不是从最深的子元素开始。

### 误区 3：阻止冒泡会影响默认行为

阻止冒泡不会影响事件的默认行为，需要使用 `preventDefault()` 来阻止默认行为。

## 总结

事件冒泡和事件捕获是 DOM 事件传播的两种机制：

- **事件冒泡**：从目标元素向上传播到根节点
- **事件捕获**：从根节点向下传播到目标元素

理解事件传播机制对于编写高效的事件处理代码至关重要，特别是在使用事件委托模式时。

---

**要点回顾**：
- DOM 事件传播分为捕获、目标、冒泡三个阶段
- 使用 `addEventListener` 的第三个参数控制使用捕获还是冒泡
- 使用 `stopPropagation()` 阻止事件继续传播
- 事件委托是利用事件冒泡实现的高效事件处理模式
