# JavaScript 原型和原型链详解

## 什么是原型？

在 JavaScript 中，每个对象都有一个特殊的属性，称为 **原型（prototype）**。原型本身也是一个对象，它包含了可以被该对象继承的属性和方法。

### 原型的作用

- **实现继承**：通过原型，对象可以继承其他对象的属性和方法
- **代码复用**：多个对象可以共享同一个原型，避免重复定义相同的方法
- **动态属性**：可以在运行时向原型添加属性和方法，所有继承该原型的对象都会自动获得这些新属性和方法

## 原型链

当我们访问一个对象的属性或方法时，如果该对象本身没有这个属性或方法，JavaScript 会沿着 **原型链（prototype chain）** 向上查找，直到找到该属性或方法，或者到达原型链的顶端（`null`）。

### 原型链的结构

- 每个对象都有一个原型（`__proto__`）
- 原型本身也是一个对象，所以它也有自己的原型
- 这样就形成了一条链式结构，称为原型链
- 原型链的顶端是 `Object.prototype`，它的原型是 `null`

## 构造函数与原型

在 JavaScript 中，构造函数是用来创建对象的函数。每个构造函数都有一个 `prototype` 属性，指向一个对象，这个对象就是通过该构造函数创建的所有实例的原型。

### 构造函数示例

```javascript
// 定义一个构造函数
function Person(name, age) {
  this.name = name;
  this.age = age;
}

// 向构造函数的原型添加方法
Person.prototype.sayHello = function () {
  console.log(`Hello, my name is ${this.name}.`);
};

// 创建实例
const person1 = new Person("Alice", 25);
const person2 = new Person("Bob", 30);

// 调用继承的方法
person1.sayHello(); // 输出: Hello, my name is Alice.
person2.sayHello(); // 输出: Hello, my name is Bob.

// 检查原型
console.log(person1.__proto__ === Person.prototype); // true
console.log(Person.prototype.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__); // null
```

## `__proto__` 和 `prototype` 的区别

- **`__proto__`**：是对象实例的属性，指向该实例的原型
- **`prototype`**：是构造函数的属性，指向该构造函数创建的所有实例的原型

### 关系图

```
person1.__proto__ → Person.prototype
Person.prototype.__proto__ → Object.prototype
Object.prototype.__proto__ → null

Person.__proto__ → Function.prototype
Function.prototype.__proto__ → Object.prototype
```

## 原型继承

JavaScript 中的继承主要通过原型链实现。当一个对象继承另一个对象时，它会继承该对象的所有属性和方法。

### 原型继承示例

```javascript
// 父构造函数
function Animal(name) {
  this.name = name;
  this.eat = function () {
    console.log(`${this.name} is eating.`);
  };
}

// 向父构造函数的原型添加方法
Animal.prototype.sleep = function () {
  console.log(`${this.name} is sleeping.`);
};

// 子构造函数
function Dog(name, breed) {
  // 调用父构造函数
  Animal.call(this, name);
  this.breed = breed;
}

// 继承父构造函数的原型
Dog.prototype = Object.create(Animal.prototype);
// 修复构造函数指向
Dog.prototype.constructor = Dog;

// 向子构造函数的原型添加方法
Dog.prototype.bark = function () {
  console.log(`${this.name} is barking.`);
};

// 创建实例
const dog = new Dog("Buddy", "Golden Retriever");

// 调用继承的方法
dog.eat(); // 输出: Buddy is eating.
dog.sleep(); // 输出: Buddy is sleeping.
dog.bark(); // 输出: Buddy is barking.

// 检查原型链
console.log(dog.__proto__ === Dog.prototype); // true
console.log(Dog.prototype.__proto__ === Animal.prototype); // true
console.log(Animal.prototype.__proto__ === Object.prototype); // true
```

## 原型链的查找机制

当访问一个对象的属性或方法时，JavaScript 会按照以下顺序查找：

1. 首先在对象本身查找
2. 如果找不到，就到对象的原型（`__proto__`）中查找
3. 如果还是找不到，就到原型的原型中查找
4. 以此类推，直到找到该属性或方法，或者到达原型链的顶端（`null`）

### 查找示例

```javascript
const obj = {};

// 访问 obj.toString()
// 1. 在 obj 本身查找 toString - 找不到
// 2. 在 obj.__proto__ (即 Object.prototype) 中查找 toString - 找到
console.log(obj.toString()); // 输出: [object Object]

// 访问 obj.nonExistentMethod()
// 1. 在 obj 本身查找 nonExistentMethod - 找不到
// 2. 在 obj.__proto__ 中查找 nonExistentMethod - 找不到
// 3. 在 obj.__proto__.__proto__ (即 null) 中查找 nonExistentMethod - 找不到
// 4. 抛出错误
console.log(obj.nonExistentMethod()); // 抛出错误: obj.nonExistentMethod is not a function
```

## 原型链的性能影响

原型链查找会影响性能，尤其是当原型链较长时。每次访问属性或方法都需要沿着原型链向上查找，这会增加时间复杂度。

### 性能优化建议

1. **避免过深的原型链**：过深的原型链会增加查找时间
2. **将常用方法定义在对象本身**：对于频繁访问的方法，直接定义在对象本身上可以避免原型链查找
3. **使用 `Object.hasOwnProperty()`**：检查属性是否存在于对象本身，而不是原型链上
4. **使用 `Object.create(null)`**：创建一个没有原型的对象，避免原型链查找

## ES6 中的类和继承

ES6 引入了 `class` 语法，使 JavaScript 的面向对象编程更加直观。虽然底层仍然使用原型链实现，但语法更加简洁清晰。

### ES6 类示例

```javascript
// 父类
class Animal {
  constructor(name) {
    this.name = name;
  }

  eat() {
    console.log(`${this.name} is eating.`);
  }

  sleep() {
    console.log(`${this.name} is sleeping.`);
  }
}

// 子类
class Dog extends Animal {
  constructor(name, breed) {
    super(name); // 调用父类构造函数
    this.breed = breed;
  }

  bark() {
    console.log(`${this.name} is barking.`);
  }
}

// 创建实例
const dog = new Dog("Buddy", "Golden Retriever");

// 调用继承的方法
dog.eat(); // 输出: Buddy is eating.
dog.sleep(); // 输出: Buddy is sleeping.
dog.bark(); // 输出: Buddy is barking.

// 检查原型链
console.log(dog.__proto__ === Dog.prototype); // true
console.log(Dog.prototype.__proto__ === Animal.prototype); // true
console.log(Animal.prototype.__proto__ === Object.prototype); // true
```

## 常见的原型链问题

### 1. 修改原型影响所有实例

当修改构造函数的原型时，所有通过该构造函数创建的实例都会受到影响。

```javascript
function Person(name) {
  this.name = name;
}

const person1 = new Person("Alice");
const person2 = new Person("Bob");

// 向原型添加方法
Person.prototype.sayHello = function () {
  console.log(`Hello, my name is ${this.name}.`);
};

// 所有实例都可以访问新方法
person1.sayHello(); // 输出: Hello, my name is Alice.
person2.sayHello(); // 输出: Hello, my name is Bob.
```

### 2. 原型链的顶端

原型链的顶端是 `Object.prototype`，它的原型是 `null`。

```javascript
console.log(Object.prototype.__proto__); // null
```

### 3. `constructor` 属性

每个构造函数的原型都有一个 `constructor` 属性，指向该构造函数。当我们修改原型时，需要注意修复 `constructor` 属性的指向。

```javascript
function Person(name) {
  this.name = name;
}

// 修改原型
Person.prototype = {
  sayHello: function () {
    console.log(`Hello, my name is ${this.name}.`);
  },
};

// 修复 constructor 属性
Person.prototype.constructor = Person;

const person = new Person("Alice");
console.log(person.constructor === Person); // true
```

## 原型链的应用

### 1. 实现继承

通过原型链可以实现对象之间的继承关系，使代码更加复用。

### 2. 扩展内置对象

可以通过修改内置对象的原型来扩展它们的功能。

```javascript
// 扩展 Array 原型
Array.prototype.sum = function () {
  return this.reduce((total, item) => total + item, 0);
};

const numbers = [1, 2, 3, 4, 5];
console.log(numbers.sum()); // 输出: 15
```

### 3. 实现对象的深度拷贝

可以利用原型链来实现对象的深度拷贝。

```javascript
function deepClone(obj) {
  if (obj === null || typeof obj !== "object") {
    return obj;
  }

  // 创建一个新对象，使用原对象的原型
  const clone = Object.create(Object.getPrototypeOf(obj));

  // 复制原对象的属性
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key]);
    }
  }

  return clone;
}

const original = { name: "Alice", age: 25, address: { city: "New York" } };
const clone = deepClone(original);

// 修改克隆对象不会影响原对象
clone.address.city = "London";
console.log(original.address.city); // 输出: New York
console.log(clone.address.city); // 输出: London
```

## 总结

原型和原型链是 JavaScript 中非常重要的概念，它们是实现继承和代码复用的基础。理解原型和原型链对于掌握 JavaScript 的面向对象编程至关重要。

### 关键点

1. **每个对象都有一个原型**：通过 `__proto__` 属性访问
2. **构造函数有一个 prototype 属性**：指向所有实例的原型
3. **原型链**：当访问属性或方法时，会沿着原型链向上查找
4. **原型继承**：通过修改原型来实现对象之间的继承关系
5. **ES6 类**：语法糖，底层仍然使用原型链实现

### 最佳实践

1. **合理使用原型**：将共享的方法和属性定义在原型上
2. **避免修改内置对象的原型**：可能会导致冲突和不可预测的行为
3. **注意原型链的长度**：过长的原型链会影响性能
4. **使用 `Object.hasOwnProperty()`**：检查属性是否存在于对象本身
5. **理解 `this` 的指向**：在原型方法中，`this` 指向调用该方法的对象

通过掌握原型和原型链的概念，你可以写出更加优雅、高效的 JavaScript 代码，更好地理解 JavaScript 的工作原理。
